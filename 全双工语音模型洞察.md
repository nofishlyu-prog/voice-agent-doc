## 什么是双工模型

### 定义

**双工模型 (Full-Duplex Model)** 是指能够**同时进行输入和输出**的语音交互模型。与传统半双工（Half-Duplex）模型不同，双工模型支持：

- ✅ **同时听和说** - 在说话的同时接收用户输入
- ✅ **实时打断** - 用户可以随时打断 AI 的回复
- ✅ **低延迟交互** - 端到端延迟 <500ms
- ✅ **连续对话** - 无需"唤醒 - 响应"模式

### 半双工 vs 双工对比

| 特性 | 半双工 (Half-Duplex) | 双工 (Full-Duplex) |
|------|---------------------|-------------------|
| **交互模式** | 你说 - 我说 (交替) | 同时说和听 |
| **延迟** | 1-3 秒 | <500ms |
| **打断支持** | ❌ 不支持 | ✅ 支持 |
| **自然度** | 机械式对话 | 接近真人交流 |
| **技术难度** | 中 | 高 |
| **代表产品** | Siri, Alexa (早期) | MiniCPM-o 4.5, Gemini Live |

### 典型对话流程对比

**半双工模式**:
```
用户："今天天气怎么样"  [说完等待]
  ↓
系统：[处理 1-2 秒]
  ↓
系统："今天天气晴朗..." [说完等待]
  ↓
用户："那明天呢" [说完等待]
  ↓
... (循环)
```

**双工模式**:
```
用户："今天天气怎么样"
  ↓
系统：[边听边准备回复]
  ↓
系统："今天天气晴朗" [用户可随时打断]
  ↓
用户："那明天呢" [直接插入]
  ↓
系统：[立即响应] "明天也是好天气"
  ↓
... (流畅连续对话)
```

### 关键技术点

#### 1. 流式处理 (Streaming)

```python
# 伪代码示例

class FullDuplexAgent:
    async def process(self):
        # 同时运行输入和输出流
        input_task = asyncio.create_task(self.input_stream())
        output_task = asyncio.create_task(self.output_stream())
        
        # 等待任一完成
        done, pending = await asyncio.wait(
            [input_task, output_task],
            return_when=asyncio.FIRST_COMPLETED
        )
        
        # 处理打断
        if input_task in done:
            # 用户说话了，停止输出
            await self.interrupt_output()
```

#### 2. 轮次管理 (Turn-Taking)

```python
class TurnManager:
    def __init__(self):
        self.floor_holder = None  # 当前发言者
        self.transition_relevance_place = None  # 转换相关位置
    
    def detect_turn_transition(self, vad_event, asr_text, llm_progress):
        """
        检测轮次转换时机
        
        判断依据:
        1. VAD 检测到用户语音
        2. ASR 识别到完整句子
        3. LLM 生成到句子边界
        4. TTS 播放到自然停顿点
        """
        if vad_event.is_user_speech:
            # 用户开始说话
            if self.floor_holder == "agent":
                # AI 正在说话，需要打断
                return "interrupt"
            else:
                # AI 未说话，让用户说
                return "yield"
        
        if asr_text.is_complete_sentence:
            # 用户说完，AI 可以回复
            return "agent_turn"
        
        return "continue"
```

#### 3. 打断处理 (Interruption)

```python
class InterruptionHandler:
    def __init__(self):
        self.tts_playing = False
        self.pending_audio = []
    
    async def handle_interruption(self):
        """处理用户打断"""
        
        # 1. 立即停止 TTS 播放
        await self.tts.stop()
        
        # 2. 清空待发送队列
        self.pending_audio.clear()
        
        # 3. 标记打断状态
        self.interrupted = True
        
        # 4. 优先处理用户输入
        await self.process_user_input()
        
        # 5. 根据上下文决定是否继续
        if self.should_resume():
            await self.resume_response()
```

#### 4. 低延迟优化

```python
# 流水线并行处理

class PipelineParallel:
    def __init__(self):
        # 创建处理管道
        self.pipeline = [
            VADHandler(),
            ASRHandler(),
            LLMHandler(),
            TTSHandler(),
        ]
    
    async def process_stream(self, audio_stream):
        """流式处理"""
        
        # 创建队列连接各个阶段
        queues = [asyncio.Queue() for _ in self.pipeline]
        
        # 启动所有处理阶段
        tasks = []
        for i, handler in enumerate(self.pipeline):
            if i == 0:
                # 第一阶段从音频流读取
                task = asyncio.create_task(
                    handler.process(audio_stream, queues[i])
                )
            else:
                # 后续阶段从前一阶段队列读取
                task = asyncio.create_task(
                    handler.process(queues[i-1], queues[i])
                )
            tasks.append(task)
        
        # 等待完成
        await asyncio.gather(*tasks)
```

### 3. LLM (Large Language Model)

**作用**: 理解上下文，生成回复

**双工优化**:
```python
class StreamingLLM:
    async def generate(self, prompt, context):
        """流式生成"""
        
        # 增量生成 token
        async for token in self.model.generate_stream(
            prompt, 
            context,
            max_new_tokens=512
        ):
            # 检测到句子边界时立即输出
            if self.is_sentence_boundary(token):
                yield self.current_sentence
                self.current_sentence = ""
            else:
                self.current_sentence += token
```

**流式 TTS 示例**:

```python
class StreamingTTS:
    async def synthesize(self, text):
        """流式合成"""
        
        # 分句合成
        sentences = self.split_sentences(text)
        
        for sentence in sentences:
            # 合成当前句
```

### 5. Turn Detection (轮次检测)

**作用**: 判断用户是否说完话

**技术方案**:
```python
class TurnDetector:
    def __init__(self):
        self.eou_model = EOUModel()  # End-of-Utterance 模型
        self.vad = VADHandler()
        self.asr = ASRHandler()
    
    async def detect_turn(self, audio_stream):
        """检测轮次"""
        
        while True:
            # 1. VAD 检测语音
            vad_event = await self.vad.detect(audio_stream)
            
            # 2. ASR 识别文本
            asr_text = await self.asr.recognize(audio_stream)
            
            # 3. EOU 判断是否结束
            eou_prob = await self.eou_model.predict(
                asr_text,
                vad_event
            )
            
            if eou_prob > 0.5:
                # 用户说完了
                return "user_turn_end"
            elif vad_event.is_silence > 2.0:
                # 静音超过 2 秒
                return "user_turn_end"
            else:
                # 用户还在说
                continue
```

```
3. 硬件加速

   - 边缘计算
```

### 2. 打断处理

**挑战**:
- 检测打断时机
- 优雅停止 TTS
- 上下文管理

**解决方案**:
```python
class SmartInterruption:
    def handle(self):
        # 1. VAD 立即检测
        if self.vad.detect_user_speech():
            # 2. 立即停止 TTS
            self.tts.stop(fade_out=True)  # 淡出停止
            
            # 3. 保存当前状态
            self.save_context()
            
            # 4. 处理用户输入
            self.process_user_input()
            
            # 5. 智能恢复
            if self.should_resume():
                self.resume_from_checkpoint()
```

### 3. 多模态融合

**挑战**:
- 视觉 + 音频 + 文本对齐
- 跨模态注意力
- 资源消耗大

**解决方案**:
```
1. 统一表示
   - 所有模态映射到同一 embedding 空间
   
2. 交叉注意力
   - Cross-Attention 机制
   
3. 高效架构
   - 共享编码器
   - 轻量级投影
```

### 4. 隐私保护

**挑战**:
- 语音数据敏感
- 云端处理风险
- 合规要求

**解决方案**:
```
1. 本地处理
   - 端侧推理
   - 数据不出设备
   
2. 联邦学习
   - 模型更新不上传数据
   
3. 差分隐私
   - 添加噪声保护隐私
   
4. 加密传输
   - TLS/SSL 加密
   - 端到端加密
```

### 核心优势分析

#### 1. 参数量优势 (7B)

**优化技术**:
- 流式 ASR (边说边识别)

- 内存限制 (6-8GB)
- 算力限制 (移动端)
- 功耗限制 (电池设备)

**优化方案**:
```
1. 量化压缩
   - INT4 量化 (7B → 3.5GB)
   - 混合精度 (关键层 FP16)

2. 模型剪枝
   - 结构化剪枝
   - 注意力头剪枝

3. 蒸馏优化
   - 从大模型蒸馏知识
   - 保持性能的同时减小规模
```
